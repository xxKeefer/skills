# Issue Template

Use this structure. Adapt sections to the task — drop what's irrelevant, add what's needed.

---

```markdown
# {conventional prefix}: {Brief Title}

## Summary

{2-4 sentences. What are we building/changing and why.}

**Spike:** #N _(if applicable)_

---

## Acceptance Criteria

- [ ] [User-visible behaviour — testable by clicking through the UI]
- [ ] [Another behaviour]
- [ ] [Error state / edge case]

---

## Reference Implementation

{Point to an existing file or pattern in the codebase that this work should follow.
Include file path and a 1-sentence description of why it's the reference.}

---

## Steps

### Step 1: {Title}

**What:** {1-2 sentences}
**Files:** {paths to create or modify}
**Done when:** {single sentence}

### Step 2: {Title}

**What:** {1-2 sentences}
**Files:** {paths to create or modify}
**Done when:** {single sentence}

---

## Out of Scope

- {Thing explicitly not being done and why}
```
